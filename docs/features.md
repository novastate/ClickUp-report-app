# Sprint Reporter

Sprint Reporter gives your team full visibility into sprint execution — from planning to retro to long-term trends. Built for teams using ClickUp who need better reporting.

## The Problem

ClickUp lacks good tools for:
- Reviewing how a sprint actually went after closing
- Tracking what didn't get done and what carries over
- Comparing team performance over time
- Getting a clear picture of scope creep during a sprint

## What the App Does

### Live Sprint View
Real-time overview during an active sprint. Auto-refreshes every 30 seconds.
- Forecasted vs completed — are we on track?
- Burndown chart against ideal line
- Scope changes — what's been added or removed since forecast
- On Track / Behind / Ahead indicator
- Capacity planning per person or team

### Sprint Report (after close)
Frozen report that preserves all data even after tasks are moved in ClickUp.
- **Unfinished tasks** shown prominently at the top with orange markers
- **Carry-over tracking** — shows which tasks were brought forward from the previous sprint
- **Workload distribution** — per person: assigned, completed, completion%, hours/points
- Scope change timeline with sprint day
- Forecast accuracy — how close was the plan?

### Trends Over Time
Team performance sprint over sprint.
- Velocity trend with average line
- Completion rate with 80% target
- Forecast accuracy trend with 90% target
- Unfinished & carry-over per sprint — are we getting better at finishing?
- Scope changes per sprint — is scope creep increasing or decreasing?
- Delta arrows on KPIs — compare recent period to earlier

### Teams & Configuration
- Support for multiple teams with individual settings
- Three metric types: task count, story points, or hours
- Capacity mode: individual, team total, or none
- Automatic sync of sprint lists from ClickUp
- Supports both day/month and month/day date formats
- Auto-close sprints when end date has passed

## Key Features

| Feature | Description |
|---------|-------------|
| Forecast baseline | Locks a snapshot at sprint start to measure against |
| Final snapshot | Captures end state of all tasks at close |
| Scope change tracking | Detects added/removed tasks automatically |
| Carry-over detection | Identifies tasks brought forward from the previous sprint |
| Unfinished highlighting | Clear visual flagging of what didn't get done |
| Workload distribution | Per-person breakdown in the retro report |
| Trend analysis | Sprint-over-sprint comparisons with delta indicators |
| Auto-refresh | Live data without manual updates |
| Auto-close | Sprints close automatically at end date |
| Multi-team | Each team has its own settings and sprints |
