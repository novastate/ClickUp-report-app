## Notes for the implementer

**`SESSION_ENCRYPTION_KEY` env var:** the test suite imports modules that require it (auth modules from Initiative 3). Always run pytest with the key set:
```bash
SESSION_ENCRYPTION_KEY=$(./.venv/bin/python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())") \
  ./.venv/bin/pytest tests/ -v
```

**Workspace-id arg value:** When adding the missing `workspace_id` positional in `create_team` calls, use `"ws_test"` for `test_*_service.py` tests. The tests don't assert on workspace_id — it just needs to be a non-empty string. Keep it consistent across all 12 call sites for readability.

**Commit cadence:** One commit per task. Conventional Commits style (`test(suite-name): describe fix`).
